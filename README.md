# Edge Z.E.R.O. — Zero-Touch Onboarding Hub

**Edge Z.E.R.O.** — это система автоматизированного развёртывания (PXE/iPXE Zero-Touch Provisioning) и оркестрации промышленных крайних серверов и узлов дорожного видеомониторинга.

## 🚀 Что было реализовано по проекту (Отчёт за 4 дня разработки)

За последние 4 дня система была полностью переработана, переведена на стек **React + TypeScript** и доведена до промышленного стандартизированного решения:

### 1. Модернизация UI/UX и стандартизация с Edge B.R.O.
* **Переход на React 19 + TypeScript + Vite + TailwindCSS:** Старый клиент полностью переписан на компонентную архитектуру с поддержкой строгой типизации.
* **Единый дизайн-код Edge Z.E.R.O.:** Внедрёна адаптивная тёмная и светлая темы, фирменный защитный логотип-айдентика, гибкая табличная сетка и плавные переходы.
* **Мультиязычность (i18n):** Полноценная поддержка **английского**, **украинского** и **русского** языков для всех вкладок, диалоговых окон, настроек и подсказок.
* **Мониторинг ресурсов в реальном времени:** В шапке приложения отображаются метрики загрузки CPU, RAM и текущая пропускная способность сетевых интерфейсов (RX/TX).

### 2. Zero-Touch PXE Provisioning & Auto-Discovery
* **Поддержка iPXE и Debian/Ubuntu:** Автоматическая генерация конфигураций `preseed.cfg` (Debian) и `user-data` (Ubuntu Subiquity) на основе MAC-адреса устройства.
* **Режим ProxyDHCP:** Добавлена поддержка работы `dnsmasq` в режиме ProxyDHCP (порт 4011), что позволяет оркестратору безопасно разворачиваться в существующих лабораторных сетях (например, под управлением MikroTik) без конфликтов IP-пулов.
* **Автоматическое обнаружение (MAC Auto-Discovery):** Парсинг журналов DHCP и обработка iPXE-циклов для мгновенного появления неавторизованных боксов в интерфейсе с возможностью их быстрой регистрации.

### 3. Real-Time логирование установки (Debian Syslog & Subiquity)
* **Debian Syslog UDP Receiver:** Реализован асинхронный UDP-сервер логов (`5140/udp`), принимающий стандартные сообщения от Debian Installer (`d-i preseed/syslog-server`) и привязывающий их к объекту `Box` в базе данных.
* **Ubuntu Subiquity Integration:** Обработка высокоуровневых событий процесса установки Ubuntu через HTTP-эндпоинт `/report`.
* **Интерактивный ConsoleDrawer:** Панель консоли с 4-этапным индикатором прогресса установки (PXE Boot → OS Install → Run Scripts → Finalizing), индикатором процента готовности и живым выводом журналов установки.

### 4. Веб-терминал SSH (Web SSH Terminal)
* **Встроенный эмулятор xterm.js:** Интегрирован полнофункциональный терминал с автоматическим масштабированием размерности (`@xterm/addon-fit`) и настраиваемой цветовой схемой.
* **Асинхронный WebSocket SSH Proxy:** Модуль бэкенда (`/api/ssh/{box_id}` + `asyncssh`) осуществляет безопасное двунаправленное проксирование SSH TTY-сессий к активным узлам прямо из браузера.

### 5. Управление профилями и визуальный конфигуратор ОС (Profiles Tab)
* **Выделенная вкладка Профилей:** Реорганизация интерфейса с выделением глобальных параметров ОС в отдельный верхнеуровневый раздел **Profiles**.
* **Визуальный конфигуратор Preseed:** Удобная настройка локали, часового пояса, раскладки клавиатуры, зеркал пакетов, NTP, Gateway, DNS и SSH-ключей.
* **Безопасное хэширование паролей:** Прозрачное автоматическое генерация SHA-512 crypt хэшей паролей root и пользовательских аккаунтов на бэкенде перед записью в пресид.

### 6. Аппаратный аудит и контроль целостности (Hardware Baseline Diff)
* **Сбор оборудования:** Скрипт постановочного аудита в `init.sh` собирает полную спецификацию устройства (процессор, ОЗУ, PCI-устройства, USB-хабы, последовательные порты `/dev/ttyS*`, сетевые карты).
* **Сравнение с эталоном (Baseline & Diff):** Визуальный модальный аудит с подсветкой добавленного, удалённого или подменённого оборудования относительно принятого эталона.
* **Фоновый мониторинг (Heartbeats & Maintenance):** Таймаут-контроллер отслеживает активные боксы и автоматически переводит недоступные узлы в статус `MAINTENANCE` с отправкой оповещений.

### 7. Интеграция с Edge B.R.O.
* **Вкладка интеграции бэкапов:** Интерактивная панель сверки узлов fleet-узлов с бэкап-оркестратором Edge B.R.O.
* **Автоматическое зачисление:** Регистрация и передача параметров аутентификации узлов в кластер бэкапов в один клик.

### 8. Безопасность, Telegram и системный аудит
* **Ролевая модель пользователей:** Разграничение прав доступа (Administrator, Operator, Viewer).
* **Персональные уведомления в Telegram:** Подписка администраторов по их `telegram_id` для получения индивидуальных уведомлений при вводе нод в строй.
* **Журналирование действий (Audit Logs):** Хранение истории действий пользователей и вывода серверных логов в базе данных с просмотром во вкладке Logs.

## 📋 Требования и быстрый запуск

### Зависимости
* **Docker** 24.0+
* **Docker Compose** 2.20+

### Запуск в среде разработки

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/masseselsev/edge-zero.git
   cd edge-zero
   ```

2. Запустите контейнеры через Docker Compose:
   ```bash
   docker compose up -d --build
   ```

3. Откройте веб-интерфейс в браузере:
   * **Frontend:** http://localhost:5555
   * **Backend API Docs:** http://localhost:7000/docs

### Данные по умолчанию
* **Логин:** `admin`
* **Пароль:** `admin123`

Если требуется сбросить пароль администратора:
```bash
docker compose exec overwatch-core python /app/create_admin.py
```
